# 腾讯商业模式评估体系 v1.0

## 使用说明
本评估体系用于季度跟踪腾讯商业模式的变化，目标是将财报信息转化为结构化判断，而不是预测短期股价。每季度按模块填写指标并评分，形成持续跟踪记录。

---

## 总体框架

评估体系分为四个核心模块：

A. 增长引擎评估（Growth Engine）  
B. 盈利质量评估（Profit Quality）  
C. 护城河评估（Moat Strength）  
D. 风险与结构变化（Risk & Regime）

每个模块包含：关键指标、观察方法、判断逻辑。

---

## A. 增长引擎评估（Growth Engine）

### A1. 用户生态增长

**关键指标：**
- 微信 MAU 增速
- 用户使用时长趋势
- 视频号活跃度

**观察方法：**
从季度财报与管理层披露中提取用户数据与趋势描述。

**判断逻辑：**
- MAU 上升 + 时长上升 → 生态健康
- MAU 持平 + 时长下降 → 警告信号
- 新功能带动使用时长 → 增长潜力

---

### A2. 业务结构增长

**关键指标：**
- 游戏收入同比增速
- 广告收入同比增速
- 金融科技与企业服务增速

**观察方法：**
拆分收入结构，比较各业务增速与占比变化。

**判断逻辑：**
- 新业务增速高于核心业务 → 结构升级
- 全线放缓 → 宏观或周期压力

---

## B. 盈利质量评估（Profit Quality）

### B1. 毛利率趋势

**关键指标：**
- 综合毛利率
- 各业务分部毛利率（如披露）

**判断逻辑：**
- 毛利率上升 → 商业模式强化
- 毛利率下降 → 竞争或成本压力

---

### B2. 经营现金流 vs 净利润

**关键指标：**
- 经营现金流
- 净利润

**判断逻辑：**
- 现金流 ≥ 净利润 → 盈利扎实
- 现金流 < 净利润 → 需关注质量

---

### B3. 费用结构

**关键指标：**
- 研发费用占比
- 销售及管理费用趋势

**判断逻辑：**
- 研发投入提升 → 投资未来
- 费用率无效率上升 → 运营压力

---

## C. 护城河评估（Moat Strength）

### C1. 用户粘性

**关键指标：**
- DAU / MAU 比例
- 留存率（如披露）

**判断逻辑：**
- 粘性提升 → 护城河强化
- 粘性下降 → 竞争压力上升

---

### C2. 生态活跃度

**关键指标：**
- 小程序交易规模
- 开发者或创作者生态指标

**判断逻辑：**
- 生态繁荣 → 网络效应增强
- 生态停滞 → 长期风险

---

### C3. 竞争态势（定性）

**观察要点：**
- 用户时长是否被其他平台分流
- 新兴平台是否侵蚀核心场景

---

## D. 风险与结构变化（Risk & Regime）

### D1. 监管变量

**观察要点：**
- 是否限制商业化能力
- 是否改变行业竞争规则

---

### D2. 资本配置

**关键指标：**
- 股票回购
- 投资与并购节奏
- 现金储备变化

**判断逻辑：**
反映管理层对长期战略与资本效率的判断。

---

## 季度评分系统

每季度为四个模块打分（-2 至 +2）：

| 模块 | 评分（-2 ~ +2） | 备注 |
|------|----------------|------|
| 增长引擎 |  |  |
| 盈利质量 |  |  |
| 护城河 |  |  |
| 风险 |  |  |

**总分区间解释：**

- +5 ~ +8 → 强势期
- +1 ~ +4 → 健康期
- -1 ~ 0 → 观察期
- < -1 → 风险期

---

## 季度执行流程

### Step 1
快速扫描财报，填写关键指标（5–10 分钟）。

### Step 2
按四个模块评分（约 5 分钟）。

### Step 3
写一句总结判断：

“本季度商业模式：强化 / 稳定 / 弱化”。

---

## 目标

通过持续季度跟踪，将主观感觉转化为结构化判断，建立长期可复用的投资分析体系。
